Install Required Software

Before we start, you need to install three free tools on your computer. These are what developers use to build websites.

1.1

Install Node.js (JavaScript Runtime)

Node.js lets your computer run the code. Think of it like installing a video player to watch movies.

  1. Go to nodejs.org
  2. Click the big green "LTS" button (LTS means "Long Term Support" - it's the stable version)
  3. Download and run the installer
  4. Click "Next" through all the prompts (default settings are fine)
  5. Restart your computer after installation
üí° How to Verify

Open your terminal/command prompt and type:

node --version

If you see a version number like v20.x.x, it's working!

1.2

Install Git (Version Control)

Git helps download and manage the code. It's like a download manager for code.

For Windows:

  1. Go to git-scm.com/download/win
  2. Download and run the installer
  3. Keep all default settings (click "Next" many times)

For Mac:

  1. Open Terminal (press Cmd + Space, type "Terminal")
  2. Type git --version and press Enter
  3. If not installed, it will prompt you to install
1.3

Install a Code Editor (VS Code)

VS Code is where you'll edit files. It's like Microsoft Word but for code.

  1. Go to code.visualstudio.com
  2. Download for your operating system
  3. Install with default settings
‚úÖ Section 1 Complete!

You now have all the tools needed. Let's create some free accounts.

2

Create Free Accounts

You'll need accounts on these platforms. All have generous free tiers.

2.1

GitHub (Code Storage)

  1. Go to github.com/signup
  2. Enter your email and create a password
  3. Choose a username (this will be visible)
  4. Verify your email
  5. Select "Skip personalization" when asked
2.2

Vercel (Hosting - Where Your Site Lives)

  1. Go to vercel.com/signup
  2. Click "Continue with GitHub"
  3. Authorize Vercel to access your GitHub
  4. That's it! Your account is ready
2.3

Supabase (Database)

  1. Go to supabase.com
  2. Click "Start your project"
  3. Sign up with your GitHub account
  4. Keep this tab open - we'll come back to it
2.4

TMDB (Movie/TV Data)

  1. Go to themoviedb.org/signup
  2. Create an account
  3. Go to Settings ‚Üí API
  4. Request an API key (select "Developer")
  5. Fill out the form (any reasonable answers work)
  6. Copy your API key somewhere safe
2.5

RAWG (Game Data)

  1. Go to rawg.io/apidocs
  2. Sign up for an account
  3. Go to your profile settings
  4. Find your API key
2.6

Google AI Studio (Gemini AI)

Optional - for AI recommendations

  1. Go to aistudio.google.com
  2. Sign in with your Google account
  3. Click "Get API key"
  4. Create a new key and copy it
üìù Keep These Safe

Create a temporary text file with all your API keys. You'll need them in Step 5. Never share these publicly!

4

Set Up the Database

The database stores all your media data. We'll use Supabase (free PostgreSQL).

4.1

Create a New Project

  1. Go back to your Supabase dashboard
  2. Click "New Project"
  3. Name it mediamind
  4. Set a database password (save this somewhere safe!)
  5. Choose a region closest to you
  6. Click "Create new project"
  7. Wait 1-2 minutes for it to be ready
4.2

Run the Database Schema

  1. In your Supabase project, click "SQL Editor" (left sidebar)
  2. Click "New query"
  3. Copy the entire contents of supabase/schema.sql from your project
  4. Paste it into the SQL Editor
  5. Click "Run" (the play button)
‚úÖ Success Check

You should see a green "Success" message. If you see red errors, make sure you copied the entire file.

4.3

Get Your Connection Details

  1. Click "Project Settings" (gear icon at bottom left)
  2. Click "API" in the sidebar
  3. Find these values:
    • Project URL - starts with https://
    • anon public - a long string of random characters
  4. Copy both to your temporary text file
5

Configure API Keys

Environment variables are secret configuration values that the app needs to connect to services.

5.1

Create the Environment File

  1. In VS Code, right-click in the file explorer (left sidebar)
  2. Select "New File"
  3. Name it .env.local (including the dot at the start)
  4. Open the file by clicking it
5.2

Add Your Configuration

Copy this template and fill in YOUR values:

# Supabase Configuration NEXT_PUBLIC_SUPABASE_URL=your_project_url_from_supabase NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key_from_supabase # AI APIs (optional but recommended) GEMINI_API_KEY=your_gemini_api_key GROQ_API_KEY=your_groq_key_if_you_have_one # Media APIs NEXT_PUBLIC_TMDB_API_KEY=your_tmdb_api_key NEXT_PUBLIC_RAWG_API_KEY=your_rawg_api_key # Google Books (optional) NEXT_PUBLIC_GOOGLE_BOOKS_API_KEY=your_google_books_key_or_leave_blank # App Configuration NEXT_PUBLIC_APP_URL=http://localhost:3000
‚ö†Ô∏è Important Security Note

Never commit this file to GitHub! The .env.local filename is already set to be ignored by git for your protection.

5.3

Save the File

  1. Press Ctrl+S (or Cmd+S on Mac)
  2. The dot after the filename should disappear

This downloads all the required packages. It may take 2-5 minutes. You'll see progress bars.

Run this command:

You should see:

  • "Local: http://localhost:3000"
  • "Ready in X.Xs"

  1. Open Chrome, Firefox, or Edge
  2. Go to: http://localhost:3000
  3. You should see the MediaMind dashboard!

You should see the MediaMind interface loading

Deploy to the Internet

This is the exciting part - putting your app live for anyone to use!

7.1

Upload to GitHub

First, we need to put the code on GitHub:

# Initialize git repository git init # Add all files git add . # Create first commit git commit -m "Initial commit" # Create repository on GitHub and link it # (Replace YOUR_USERNAME with your actual GitHub username) git remote add origin https://github.com/YOUR_USERNAME/mediamind.git # Upload to GitHub git push -u origin main

If it's your first time, it may ask for login. Use your GitHub credentials.

7.2

Connect to Vercel

  1. Go to vercel.com
  2. Click "Add New..." ‚Üí "Project"
  3. Find your mediamind repository and click "Import"
7.3

Configure Environment Variables on Vercel

  1. On the project configuration page, look for "Environment Variables"
  2. Add each variable from your .env.local file:
    • Copy the name (left side of =)
    • Copy the value (right side of =)
    • Click "Add"
  3. Repeat for all variables

Add all your API keys in the Environment Variables section

7.4

Deploy!

  1. Click "Deploy" button
  2. Wait 2-3 minutes for the build to complete
  3. You'll get a URL like https://mediamind-xyz123.vercel.app
üéâ You're Live!

Your MediaMind app is now on the internet! Click the URL to see it. Share this URL with friends!

Set Up a Custom Domain (Optional)

  1. In your Vercel project, go to "Settings" ‚Üí "Domains"

  3. Follow the DNS instructions Vercel provides

Troubleshooting Common Issues

"npm install" fails

  • Make sure Node.js is installed: node --version
  • Try deleting the node_modules folder and run again
  • Use npm install --legacy-peer-deps

"Cannot find module" errors

  • You missed the npm install step
  • Or installation was incomplete - run it again

Environment variables not working

  • Make sure file is named exactly .env.local (with the dot)
  • Restart the server after changing it
  • No spaces around the = sign

Database connection fails

  • Double-check your Supabase URL and anon key
  • Make sure you ran the SQL schema
  • Check that Row Level Security policies are set (in schema)

Changes not showing on deployed site

  • Push changes to GitHub: git push
  • Vercel automatically redeploys on every push
  • Check the "Deployments" tab on Vercel for status
